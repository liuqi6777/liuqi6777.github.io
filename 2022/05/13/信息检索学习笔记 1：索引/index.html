信息检索学习笔记 1：索引

前言

最近在看信息检索领域预训练模型的一篇综述，感觉基础还是有些欠缺，想快速补一下信息检索领域的基础知识。于是找到了《Introduction to Information Retrieve》这本书开始自学。虽然这本书成书时间较早，现代的融合深度学习的检索技术并未涉及，但仍不失为一本经典入门著作。作者是斯坦福教授 Manning，在斯坦福也开了对应的课程 CS276，可惜没有公开上课视频。不过结合书和 slides，再做一下课程项目，自学倒也问题不大。

信息检索

信息检索是从从大规模非结构化数据集中找出满足用户信息需求的资料的过程。

在信息检索任务中，我们通常有若干文档（document）。指检索系统的检索对象，可以是一条记录、一篇文章或一个网页。在不同的检索任务中我们需要选择不同的索引粒度。所有的文档组成文档集（collection），或称语料库（corpus）。在 ad hoc 检索（ad hoc retrieval）任务中，用户的信息需求通过查询（query）提交给系统，系统从文档集中返回与其相关（relevant）的文档。

倒排索引

对于检索任务，如何判断查询词和文档是否相关？最简单也是最直接的方法，就是判断某个文档中有没有该查询词。举个例子，假设有文档集

doc 1: I like NLP.

doc 2: I like Information Retrieval.

doc 3: Do you like NLP?

给定查询 Retrieval，那么显然，返回的相关的文档是 doc 2。这便是信息检索最简单的思想。为了达到这个目的，我们就需要事先知道每个文档里面有哪些词项（term）出现。自然地，如果我们知道一共有哪些文档、哪些词项，便可以构建一个 0-1 矩阵：

在这里，我们将每个“词”当作一个“词项”（词项通常可以用词表示，但事实上，词项还可能是词组），并忽略了标点符号。这个矩阵成为词项-文档关联矩阵，其行代表每个词项对应的文档向量，表示词项在哪些文档中出现或不出现；其列来表示每个文档对应的词项向量，表示文档中哪些词项出现或不出现。

有了这样一个矩阵，我们便可以得到词项在哪些文档中出现，并且可以通过布尔操作（AND、OR、NOT）轻松地完成“多个查询词同时出现的文档”等操作（AND 操作）。

例如：I AND NLP 的结果为 doc 1，Information OR you 的结果为 doc 2 和 doc 3。

但是，当文档集变大，其中包含的词项将会非常多，词项-文档关联矩阵将占用巨大的存储空间。考虑一个文档数量为 100 万，词项数为 50 万的文档集，那么这个矩阵共有 5000 亿个元素，远远大于计算机的容量！不难发现，这个庞大的矩阵实际上高度稀疏（sparse），即大部分元素都是 0 而只有很少元素是 1。假设每个文档有 1000 个单词，那么矩阵中最多也就只有 1000 x 1000000 个 1，只占了元素总数的 0.2%。

由此引出了信息检索的一个核心概念——倒排索引（inverted index）。

倒排索引的基本思想为只记录词项出现的文档。如图所示，左部称为词项词典（dictionary）。每个词项都有一个记录出现该词项的所有文档的列表，表中的每个元素记录的是词项在某文档中的一次出现信息，称为倒排记录（posting），词项对应的整个表称为倒排记录表（posting list）。所有词项及其对应的所有倒排记录表共同组成倒排索引。

建立倒排索引

建立索引的主要步骤如下：

  • 收集需要建立索引的文档
  • 将每篇文档转换成一个个词条（token）
  • 进行预处理，产生归一化的词条作为词项
  • 对所有文档按照其中出现的词项做索引构建（indexing）

我们分两部分进行讨论，前三步确定词项集合，第四步构建索引。

词项集合的确定

词条化（tokenize）：定义好文档单位之后，词条化是将给定的字符序列根据一定的规则（例如根据空格）拆分成一系列词条（token）的过程。在这个过程中，可能会同时去掉一些特殊字符，如标点符号等。在非严格的情况下，词条往往和词项或词通用。然而，有时我们需要对词条和词条类进行严格的区分。

去除停用词（stop words）：在一些情况下，部分常见词在检索时价值不大，需要从词表中删去。这些词称为停用词。一个常用的生成停用词表的方法就是将词项按照在文档集中出现的频率从高到低排列，选择那些语义内容与文档主题关系不大的高频词作为停用词。常见的停用词包括连词、介词等，例如中文连用词中的“和”、“并且”等。

词条归一化（normalization）：词条归一化是将看起来不完全一致的多个词条归纳成一个等价类， 以便在它们之间进行匹配的过程 。例如统一大小写、构建同义词表等等。
